- **The markdown-evidence guard judges what a verifier READS, not what it says (BG0264, round-1 repair).** The first version judged the target tokens as written and was defeated three ways: a directory glob whose every file is prose (`grep "x" docs/*`), a flag read as the pattern (`grep -c "x" a.md`), and a bare recursive directory. It now splits flags from the pattern, expands globs, walks a directory under `-r` and drops it without one, and keeps the written tokens as a floor so the verdict does not depend on the filesystem the lint ran on. The earlier claim that the surviving mutant was equivalent was false and is withdrawn.
