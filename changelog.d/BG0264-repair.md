- **The markdown-evidence guard walks the way the runner walks (BG0264).** Four versions of this guard were defeated in turn, each by re-stating a rule instead of deriving it: the target tokens as written, then a flag-aware split the runner does not perform, then a non-recursive drop this DSL never does, and finally an `rglob` walk that saw hidden, gitignored and symlinked files `rg` refuses to read - so one dotfile made an all-prose directory read as mixed and a markdown-only verifier passed. The parse now comes from `_build_command` and the walk from `rg --files`, so the guard and the runner cannot disagree about what a verifier reads.
