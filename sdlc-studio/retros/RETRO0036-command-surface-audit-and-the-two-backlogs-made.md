## What went well

- **The audit found its own gap immediately.** On first run `command_audit` reported 1 undocumented
  script - itself - which is exactly the discipline it exists to enforce; documenting it cleared it.
- **The tool turns a one-time audit into a re-runnable guard.** Rather than hand-auditing the
  surface once, the enumeration + drift + tooling check is a script the gate can run, so the surface
  cannot silently drift again. The 5 help-only commands are a real, recorded finding for slice 2.
- **The independent review caught three real defects the suite missed.** Dani Okafor flagged: the
  hint recomputing a full-repo scan twice (~1.4s on the most-run command); parked (Deferred/Blocked)
  items falsely nudged as awaiting refinement; and a `--write` audit doc certifying "every tool
  runs" without having checked. All three fixed, re-review clean.

## What was hard / what stalled

- **The awaiting-count semantics needed care.** "Awaiting refinement" is NOT the same set as
  reconcile's `undecomposed` (which exempts intake, because it guards accepted-but-undecomposed).
  Here intake (Proposed/Draft/Open) IS awaiting - but a PARKED item (Deferred/Blocked) is not. Two
  similar-looking predicates, different exemptions; getting the exemption wrong nagged the operator
  on parked work. The review caught it.
- **A generated doc must not out-run its evidence.** The audit doc certified tooling health it had
  not measured (the tool check is opt-in). A persisted claim is worse than none; the fix threads a
  `checked` flag so the doc only certifies what it verified.

## Lessons

- A predicate that looks like an existing one may need DIFFERENT exemptions - do not copy the
  exemption set. `discovery_awaiting` and reconcile's `undecomposed` both find "childless discovery
  items", but the first must count intake (it awaits refinement) and skip parked, while the second
  skips intake. Name the exemption from the predicate's own purpose, not from the neighbour's.
- A generated artefact must only assert what it verified. The audit doc claimed "every tool runs"
  while the tool check was off by default; a persisted, unverified certification is a lie with a
  long half-life. Thread a `checked` flag and phrase the claim from it.
- Compute-once, read-many on a hot path: the hint recomputed a full-repo scan the dashboard had
  already done. When two callers in one command both need an expensive value, compute it once and
  thread it, or the most-run command pays double.
