## Delivered

- **EP0034 - the schema-v4/config opt-in gate (RFC0040 keystone).** `two_backlog_enforced(root)`
  reads `two_backlog.enforce`, default OFF. The four hard gates consult it: G1 plan-refusal (US0126),
  G2 derived terminal status (US0127), `undecomposed` drift and the CR-creation Size demand (US0128).
  An existing project keeps its old flow until it opts in - the precondition for a safe breaking
  release. This repo dogfoods it on.
- **EP0035 - the `refine` command (RFC0039 keystone, and the RFC0040 migration tool).** `refine
  apply` decomposes a request into an epic + stories with the links wired (US0129); `refine show`
  surfaces the request and confirms it's refinable (US0130); it moves the request to its working
  status and raises open questions for a Three-Amigos consult (US0131). The hand-decomposition
  (CR0271 -> EP0033) is now a command.

## What went well

- **The independence gate caught TWO real bugs this sprint that would have shipped.** In Epic A, the
  reviewer found `new_batch` never threaded the target root, so a batch create read enforcement from
  the CWD not `--root` - an agent refining one project from inside another would be wrongly refused.
  In Epic B, it found `refine` was not atomic: a bad story title left an orphan epic on disk, and in
  the unenforced projects refine is the migration tool FOR, `undecomposed` is off and the orphan
  declares no link - so reconcile flagged NONE of it, while the CLI printed "refused". Both were
  fixed and re-approved. author != reviewer is the single most valuable discipline this project runs.
- **The opt-in gate is the un-break switch.** Default-off, read via `project_override` (own file
  only, so no defaults-merge can flip it on globally), it makes the whole two-backlog + sizing model
  safe to ship to existing projects - which is what RFC0040 said the release needs.
- **refine closed the loop on its own origin story.** The command exists because this project kept
  hand-decomposing CRs into epics; refine's own epic (EP0035) was the last one done by hand.

## What was hard / what stalled

- **Gating a live workflow rippled through the tests that assumed it was always on.** Making the gates
  default-off meant the two-backlog test fixtures had to opt in explicitly (`_enforce`), and new
  unenforced-path tests were added at each gate. Contained to test_two_backlogs.py this time.
- **refine's atomicity was subtle.** Input validation was split between `parse_story_spec` (points,
  up front) and `artifact.new` (titles, mid-loop), so the docstring's "validates everything before
  minting" was only half true. The fix - validate every title up front, plus a rollback guard - is
  the general lesson: a create loop is only atomic if EVERY input is checked before the first write.

## Lessons

- A gate on a live workflow must default OFF for existing projects and be opt-in, or an upgrade
  breaks them. `two_backlog_enforced` (default off, one config line to turn on) is the pattern: ship
  the discipline, but let each project adopt it when ready, and never through a defaults-merge that
  flips it on globally.
- A multi-step create is atomic only if EVERY input is validated before the first write. refine
  validated points up front but titles mid-loop, so a bad title orphaned an epic; the fix validates
  all titles (and the request's Status line) up front and rolls back on a residual IO error. Split
  validation is a half-promise of atomicity.
- The independence gate earns its cost on exactly the bugs the author cannot see: both this sprint's
  blocking bugs were invisible to the author (one a cross-repo CWD read, one an orphan invisible to
  reconcile in the very projects the tool targets), and both were caught by a reviewer attacking the
  claim rather than re-reading the code.
