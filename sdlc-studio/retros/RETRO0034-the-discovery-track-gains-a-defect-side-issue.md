## Delivered

- **US0137 (3pt) - Issue joins the discovery vocab.** New `is_discovery(type_)` (RFC/CR/Issue) is
  the one predicate every backlog-side gate consults; `is_request` stays the narrow RFC/CR set
  (`refine`'s domain). `plan` (G1) and `status backlog` now treat an Issue as Discovery.
- **US0138 (3pt) - the Issue artefact type.** `issue` registered in `ARTIFACT_TYPES` (prefix `IS`),
  the id regex, the status vocab (Open -> Triaging -> Triaged -> Resolved), a body carrying Size +
  Severity + no points, an index template, and `artifact.py new --type issue`. Validates and
  reconciles clean.
- **US0139 (5pt) - the triage command.** `triage.py apply|show` decomposes an Issue DIRECTLY into
  bugs (one level - a bug is already the delivery unit), reusing refine's atomic mint + rollback,
  with an up-front dry-run pre-flight of every bug so a bad breakdown fails empty.
- **US0140 (3pt) - the triage gates.** `reconcile` flags an accepted childless Issue as
  `undecomposed` (naming triage), and an Issue's Resolved is derived from its bugs (G2).
- **US0141 (3pt) - the docs.** `help/issue.md`, `help/triage.md`, the scripts catalogue entry, the
  README lines, the CHANGELOG entries.
- **US0142 (2pt) - refine show on a decomposed request (CR0275).** `refine show` now accepts an
  already-decomposed request and lists its epics (steering to `add`), staying read-only; `apply`
  keeps its strict precondition.

## What went well

- **The refine machinery paid off a second time.** triage reused refine's atomic mint + rollback
  shape almost wholesale; the only structural difference (one level, not two) was small. Building
  refine first made triage cheap.
- **The independent review earned its keep - again.** Dani Okafor (author != reviewer) found a real
  BLOCKER the whole test suite missed: on schema v3 a Low-severity triaged bug was folded into a
  finding-consolidation CR, so triage would mis-parent that CR as the Issue's child (and could
  delete a shared CR on rollback). The dogfood repo is v2, so nothing local exposed it - only an
  adversary who thought to run the v3 path did.
- **The fix was a clean bypass, not a special case.** `artifact.new(..., consolidate=False)` gates
  all three finding-noise controls for a DELIBERATE decomposition unit, and the preflight passes the
  same flag so the two paths cannot diverge. A defence-in-depth guard refuses to wire a non-bug.

## What was hard / what stalled

- **The v3 divergence was invisible from v2.** The pre-flight-mirrors-the-real-mint invariant held
  on v2 but silently broke on v3, because the dry-run and the real mint took different branches only
  under the v3 finding-noise controls. A pre-flight is only as honest as its fidelity to the real
  path - and that fidelity was schema-conditional. The lesson: when a dry-run is the safety
  mechanism, test it on every schema the real path branches on.
- Nothing else stalled - the vocab flips (`is_request` -> `is_discovery`) were mechanical once the
  call-site census was done, and the review confirmed none was missed or wrongly flipped.

## Lessons

- A dry-run pre-flight is only a safety net if it takes the SAME branch as the real mint on every
  configuration the real mint branches on. Here the real `artifact.new` routed a v3 Low-severity
  finding through consolidation while the dry-run did not, so triage's pre-flight passed and the
  real mint corrupted a CR. Test the pre-flight on each schema/config the create path forks on, not
  just the default.   <!-- record it: lessons add (project tier). Promote with --global only what generalises beyond this repo -->
- Extract a shared definition before the second caller: the `Parent:`/`Decomposed-into:` link
  writers moved to `lib.sdlc_md` (`insert_after_status`/`write_decomposed`) the moment triage became
  refine's second caller, rather than copied - LL0016 applied prospectively for once, not after the
  drift.
