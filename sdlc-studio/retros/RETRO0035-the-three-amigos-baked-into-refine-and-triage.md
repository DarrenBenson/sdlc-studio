## Delivered

- **US0143 (5pt) - the shared resolver.** `persona_resolve.consult` / `amigo_panel` / `seat_name`
  resolve a ceremony's Three-Amigos panel to the actual named seats, lead-first, framed with each
  seat's review render; a `panel` CLI exposes it.
- **US0144 (3pt) - refine bakes it in, engineering-led.** `--question` items route to the resolved
  seats; the result carries the named panel, not bare role strings; `--skip-personas` degrades.
- **US0145 (3pt) - triage bakes it in, QA-led.** Same, qa leading (is it reproducible? what is the
  real defect?).
- **US0146 (3pt) - record the consult.** The request/Issue gets a `> **Consulted:**` line and an
  `## Amigo Consult` section listing the questions - an audit trail, written idempotently.
- **US0147 (3pt) - degradation + independence.** No seats -> shipped defaults; a broken project
  seat -> hard error before anything is minted (fail-empty); `--skip-personas` byte-equivalent; the
  independence floor (a seat never signs off its own work) carried in the framing.
- **US0148 (3pt) - the docs.** reference-workflow-personas (the refine/triage rows + subsection),
  the script catalogue entries, help/issue + help/triage, the CHANGELOG.

## Blocked / deferred

- Nothing blocked. RFC0039's remaining scope is the command-surface / doc rewrite (CR0272, area e);

## What went well

- **The machinery was already there.** `resolve_consult` / `frame` had existed since the seats/amigos
  convergence but had NO callers - the sprint was mostly wiring, not building. The build was cheap
  because the hard part (declared-role resolution, the review render, the independence floor) was
  done; this sprint gave it a reason to run.
- **refine and triage share one consult definition.** One `consult()` both call, so they resolve the
  same seats the same way - only the panel order (who leads) differs. No second implementation to
  drift.
- **The independent review found a real correctness bug the suite missed.** Dani Okafor (author !=
  reviewer - and, fittingly, the very seat this feature resolves) constructed a multi-line question
  that injected a fake `##` heading into the audit section, which then broke idempotency
  (duplicating on re-run). Also flagged 15KB of unused persona framing in every JSON result. Both
  fixed; re-review clean.

## What was hard / what stalled

- **The audit-section writer trusted its input.** `record_consult` rendered question text verbatim
  into a `##`-bounded section, so a question carrying a newline + heading marker could inject a
  section the idempotency regex then split on. The fix (collapse each question to one line) is
  small; the lesson is that a section delimited by a structural marker must sanitise any user text
  written inside it, or that text can forge the delimiter.
- Nothing else stalled - the result-shape change (`amigo_roles` -> `consult`) touched one stray test
  beyond the new ones, caught immediately.

## Lessons

- Text written into a structurally-delimited section must be sanitised against the delimiter. The
  consult audit section is bounded by `^##`; a question carrying `\n## ...` forged a heading inside
  it, breaking the in-place idempotent update. Collapsing user text to a single line before
  rendering closes it. Any "write user input into a markdown section" path has this shape.
- Wiring dormant machinery is a real deliverable. `resolve_consult`/`frame` existed and were correct
  but callerless since the seats convergence; the value was latent until a ceremony ran it. Prefer
  wiring an existing, tested primitive to building a parallel one - the sprint was small precisely
  because the primitive was already sound.
