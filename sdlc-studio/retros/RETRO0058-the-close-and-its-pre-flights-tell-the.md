## Blocked / deferred

## What went well

- **The pre-flight paid for itself inside the same sprint.** Its first real use listed all six
  outstanding prerequisites for this run's own close in one pass. Under the old behaviour that
  was six invocations, each paying a full gate run.
- **BG0216's fix was proven on the real case, not a fixture.** The workaround recorded in
  RETRO-0057 was undone: the exact wording that deadlocked that close is back in the lessons log,
  and the lane is green with no reword.
- **A sweep found the already-delivered work rather than rebuilding it.** CR0342 was complete and
  unrecorded; checking before building saved the whole unit, and a sweep of all 26 open CRs
  confirmed it was the only one, so the backlog estimate held.

## What was hard / what stalled

- **I planned around a constraint that no longer existed.** The approved plan made CR0342 the
  first item because refine output supposedly could not pass the gate. It had been fixed months
  of commits earlier. The plan's own founding premise was stale, and only checking the code
  caught it.
- **Three shipped tests were passing BECAUSE of BG0214.** Their fixtures closed artefacts the
  real run refuses, and went green only because no gate was consulted. A defect can hold tests
  upright indefinitely; fixing it is what makes them fail.
- **Two of my own new fixtures were wrong in ways that mattered**, both self-caught by running
  them: a temp workspace with no `review.two_role_after` skipped the entire evidence/sign-off
  branch, so the sign-off tests passed for the wrong reason; and the batch named story ids with
  no artefacts behind them.
- **Serial blocker discovery cost me three commit attempts** in one commit (style, then
  conformance, then a missing `Refs:` trailer), each found only after clearing the last. The
  commit path has the same disease this sprint cured in the close path.

## Lessons

- Check whether the work is already done before building it. A request left open is not evidence
  that it is outstanding, and the cheapest delivery is discovering the thing already shipped.
- A plan's premises decay. Re-verify the constraint that determines the ORDER of a plan before
  following it, because a stale premise silently costs the whole sequence.
- When a defect is fixed, the tests that were passing because of it will fail. Those failures are
  the fix working, not a regression - read each one before changing it, and correct the fixture
  rather than the assertion.
- A fixture missing a config key can silently disable the whole branch under test. If a test
  passes, mutate the code it names and confirm it fails for the reason claimed.
- A pre-flight is worth building where a gate is a chain: the information is nearly always
  available before the first attempt, and serial discovery reads as the tool moving the goalposts.
