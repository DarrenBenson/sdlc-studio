# RETRO-0032: refine builds refine: using the tool to add its own incremental-decomposition mode

> **Date:** 2026-07-15
> **Batch:** US0132, US0133 (EP0036, closing CR0274)
> **Goal:** done
> **Delivered:** 2 / 2   **Blocked:** 0

## Delivered

- **US0132 (3pt) - `refine add` core.** Appends a further epic + stories to an already-decomposed
  request; the `Decomposed-into:` write is append-only (an earlier slice is never lost). Extracted
  the shared atomic mint+wire core `_decompose`, so `apply` and `add` differ only in their
  precondition and whether the request's status is moved.
- **US0133 (2pt) - `refine add` safety.** `add` requires the request BE decomposed already (the
  inverse of `apply`); the append de-dupes by normalised id, order-preserving; a bad title or a
  mid-create failure mints nothing (shares `apply`'s up-front validation + rollback).

## Blocked / deferred

- Nothing blocked. The friction this sprint fixed was itself the finding of the last one: RFC0040's
  and RFC0039's later slices can now be wired with `refine add` rather than by hand.

## What went well

- **The sprint dogfooded `refine` for the first time, and used it to build refine's own missing
  feature.** `refine apply` decomposed CR0274 into EP0036 (0 drift, links resolved both ways, CR0274
  moved to In Progress) - then US0132/US0133 built `add`. The last hand-decomposition trained the
  tool that ends hand-decomposition; this sprint the tool decomposed its own next feature.
- **CR0274 closed by its own G2 gate**, like CR0271 before it: with EP0036 Done, the enforced repo
  derived CR0274 -> Complete. The request that asked for `refine add` was refined by refine and
  closed by the gate.
- **The independent review found NO defects** - the first refine review this arc to come back clean.
  That is the pay-off of the two prior rounds: `_decompose`/`_write_decomposed` reuse the core the
  earlier reviews hardened (up-front validation, rollback, atomic_write), so the new surface
  inherited the fixes rather than re-introducing the bugs.

## What was hard / what stalled

- **A self-inflicted `import re` miss** - `_DECOMPOSED_LINE_RE` used `re` before it was imported;
  caught immediately by the RefineTests, not shipped. A reminder that a new module-level regex needs
  its import, and that the tests are the backstop for it.
- Nothing else stalled. A small (5-point), well-scoped sprint on a hardened base.

## Lessons

- Reusing a hardened core is how a new feature inherits fixes instead of re-introducing bugs. `add`
  got apply's up-front validation, rollback and atomic_write for free by sharing `_decompose`, and
  the review found nothing - where the two earlier refine reviews each found a real bug. Extract the
  shared core BEFORE adding the variant.
- Fix the friction the last retro logged, and log the new friction you hit. This sprint delivered the
  `refine add` that RETRO0031 flagged as a blocker, and it surfaced its own tiny friction (the
  missing import) - caught by tests, not worth a ticket, but named here.
