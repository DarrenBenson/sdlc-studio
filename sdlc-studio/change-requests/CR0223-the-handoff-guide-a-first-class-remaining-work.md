## Summary

v4.1 candidate, from an operator-reviewed vendor white-paper comparison (closed-platform agentic SDLC) (their step 5 hands humans 'a granular project guide of remaining work'). An agentic epic/sprint run that stops - goal reached, budget spent, or blocked - should close by GENERATING a handoff artefact: what was delivered with evidence links, what remains with per-item pointers (file, AC, failing check), which items suit copilot-assisted completion vs human judgement, and the open decisions. Today the tail lives across hints, decisions.md and the retro; a human joining after an unattended run has no single 'here is where you pick up' document. Natural home: a workflows/ or retros/ companion generated by the sprint close.

## Acceptance Criteria

- [ ] An agentic run that ends short of the goal generates a handoff artefact naming each remaining item with its pointer (file/AC/check) and a suitability tag (copilot-tail vs judgement)
- [ ] The handoff is tool-created (artifact machinery), linked from the retro, and the next sprint plan reads it as an input
