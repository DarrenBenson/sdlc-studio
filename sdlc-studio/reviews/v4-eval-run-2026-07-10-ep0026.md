## The bug the run caught (BG0099)

Scenario 02's worker, running the greenfield PRD -> epic -> story flow on a fresh v4-DEFAULT
project (`init` seeds `schema_version: 3`), hit a hard failure: `artifact new/batch --type
story --epic EP-<ulid>` reported "epic not found" because `_find_epic` resolves the parent
with `p.stem.split('-')[0]`, which yields `EP` for a v3 ULID filename. Story-to-epic wiring
is broken on the default schema - the flagship greenfield flow. The worker diagnosed it at
source, recovered by pinning the fixture to the fully-supported v2 era (logged as a decision),
and completed the scenario; the orchestrator independently reproduced and filed it as
**BG0099** (High), rostered into EP0028. No unit test covered story-under-v3-epic creation -
this is precisely the behavioural gap the eval tier exists to close.
