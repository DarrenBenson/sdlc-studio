## Method

The two-Claude loop per `evals/README.md`: one fresh worker session per scenario against a
scratchpad fixture (the candidate skill = this repo's `.claude/skills/sdlc-studio`,
read-only), then one fresh grader session per transcript, graded item-by-item against the
scenario's expected/forbidden behaviours with the scenario's grading notes applied.

Method deviations (recorded, judged acceptable by each grader):

- Workers ran as subagents given the skill DIRECTORY as their installed-skills location
  rather than a harness-level registration, so scenario 01's EB1 was graded as "skill
  content discovered and followed without the user naming it or typing a slash command".
- Transcripts are worker-authored action logs, not raw tool-call traces; graders
  corroborated the logs against the on-disk artefacts (and for scenarios 02/03/04 verified
  the artefacts/fixture state themselves).
