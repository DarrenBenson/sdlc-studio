# Benchmark v2 calibration re-spike (2026-07-08)

> Per [protocol-v2](protocol-v2.md): the N=1 calibration spike over the two new Tier-1
> fixtures, three arms, run before any N=5 spend. Published regardless of outcome. Raw rows:
> `tools/bench/results/runs.jsonl` (`run_id: v2n1`); metrics captured via
> `transcript_metrics.py` (`metrics_source: parsed`, no hand-typed figures).

**Arms:** A = sdlc-studio pipeline (judgement-scaled, as v1) · B = plain Claude Code + the
signed-off baseline CLAUDE.md · R = pipeline + model-tier routing (two-phase: a planning
pass creates a story whose ACs pin the spec interactions, `route.py pick` selects the
delivery tier, an independent critic reviews before done). **Models:** arms A/B and all
arm-R planning/critic passes pinned to claude-sonnet-5 (matching v1); arm R delivery
workers ran at the routed tier (`arms/routed_config.yaml` map). Workspaces environmentally
isolated (arm B contained no skill at all).

## Headline findings

1. **The redesigned fixtures discriminate.** `multifile-notify-digest` produced real defect
   escapes: arms A and B both shipped a digest flush that ignores quiet hours (SPEC R5) and
   declared done with 16-17 green visible tests; the held-back suite caught both (7/8). The
   v1 finding ("no fixture could differentiate") is resolved - calibration verdict:
   **fit for N=5.**
2. **Arm A behaved like arm B, again - and both escaped identically.** As in v1, the
   judgement-scaled pipeline arm declined the full pipeline on these tickets ("judged a full
   PRD/epic/story pipeline disproportionate") and implemented directly. Its defect profile
   matched the baseline's exactly. The honest read: the pipeline's value is not delivered by
   *having the skill available* - it appears only when the process is actually driven.
3. **Arm R - the only arm forced through a planning pass - was the only arm with zero
   escapes.** Its planner wrote story ACs that explicitly pinned the quiet-hours x flush,
   throttle-counting and unsubscribed interactions BEFORE implementation; the delivery
   worker then implemented against that contract and passed 8/8 hidden tests. **Honest
   attribution: the win came from the mandated spec-extraction step, not from the model
   tiers** - on notify-digest the routed delivery ran on the same model as arms A/B (the
   tier map resolves medium to the very model A/B were pinned to, so tiers cannot explain
   the win). One residual confound: arm R also carries a critic pass, and planner-vs-critic
   contributions are not disentangled at n=1 - though the planner's ACs pinned the
   quiet-hours interaction BEFORE any code existed, which makes the planning step the
   likelier cause. Routing earned its keep separately, on cost (finding 5).
4. **Auditability graded a real gradient on the harder fixture:** R 1.0 > B 0.8 > A 0.6.
   Arm A's auditor could not cite any check for the quiet-hours behaviour (its own escape,
   independently rediscovered by the auditor) and its throttle evidence failed the mutant
   check. On ledger-drift all arms scored 1.0 - at that difficulty every arm produced
   well-evidenced work. Reviewer-independence (descriptive, weight 0): NO workspace carried
   review evidence - including arm R's, whose critic verdict lived in the orchestration
   transcript, not the workspace. Fix for N=5: the routed protocol should record its critic
   verdict into the workspace (`critic.py record`), where it belongs anyway.
5. **Cost:** the routed pipeline cost 2.1-3.1x the baseline's tokens per fixture
   (122,819/57,763 = 2.13x on notify-digest; 142,803/46,672 = 3.06x on ledger-drift -
   planning + delivery + critic vs a single pass). The overhead exceeded the tier savings
   on ledger-drift even though delivery itself was 0.25x. Routing's economics should
   improve when one plan fans out to many units; at single-ticket scale, quality (not
   cost) is what the pipeline buys.
6. **Rework: 0/6.** No run in any arm needed a fix-forward correction after a
   verified-done claim (deliberate mutation-checks counted as verification practice, per
   the carried-forward v1 definition; several runs made multiple verification passes).

## Caveats

- **n=1 per cell.** Read direction, not magnitude; error bars come with N=5.
- **Arm A vs R differ in instruction, not just machinery** - R's CLAUDE.md mandates the
  planning step where A's leaves judgement. The R-vs-B comparison is therefore the cleaner
  "pipeline value" signal; A-vs-B measures the *default* posture. Protocol v2's arms are
  pre-registered as-is; any wording change is a v3 matter.
- **Grader fix, disclosed:** the Class-T quote validator initially rejected
  logically-verbatim quotes that crossed hard-wrapped lines or markdown emphasis, depressing
  T scores for every arm (pre-fix scores: nd A 0.2 / B 0.4 / R 0.8; ld A 0.5 / B 0.833 /
  R 0.5). Fixed in `audit_quiz.py` (whitespace/emphasis-normalised matching, regression
  test added), same collected answers re-graded. Class-D scores were unaffected.
- Ledger-drift produced no escapes in any arm at n=1 - it may sit below the escape
  threshold for current models; its value here was exercised through Auditability and the
  spec-drift check. Keep, but expect notify-digest to carry the defect-escape signal.

## N=5 go/no-go recommendation

**Go**, per the pre-registered plan (Tier 1 N=5, Tier 2 N=2, cut order stands), with two
harness notes carried in: record arm R's critic verdict into the workspace, and keep the
fixed quote validator. The v4 premise now has its first supporting datapoint: the pipeline's
mandated planning discipline prevented the exact defect class both unstructured arms
shipped, and the Auditability metric separated the arms on evidence quality rather than
artifact presence.
