# Benchmark v2 measured run: Tier 1 N=5 (2026-07-08)

> Per [protocol-v2](protocol-v2.md): the pre-registered measured run - Tier 1 at N=5 per
> cell (2 fixtures x 3 arms = 30 runs), Tier 2 via pre-declared cut #1 (the v1 spike rows
> stand as the scale-down control, n=1, never pooled with Tier 1). Published regardless of
> outcome. Raw rows: `tools/bench/results/runs.jsonl` (`run_id: n1`-`n5`); all metrics
> captured via `transcript_metrics.py` (`metrics_source: parsed`); every run's audit pass
> graded by `audit_quiz.py` with mutant-checked citations. Calibration rows (`v2n1`) are
> published but excluded from every aggregate below, per protocol.

**Arms:** A = sdlc-studio pipeline (judgement-scaled) · B = plain Claude Code + the
signed-off baseline CLAUDE.md · R = pipeline + model-tier routing with a mandated planning
pass (planner writes story ACs pinning spec interactions, `route.py pick` selects the
delivery tier, an independent critic records its verdict into the workspace).
**Models:** arms A/B and all arm-R planner/critic passes pinned to claude-sonnet-5; arm R
delivery workers ran at the routed tier (`arms/routed_config.yaml`: tiny=claude-haiku-4-5,
medium=claude-sonnet-5). Workspaces environmentally isolated (arm B contained no skill).

## Headline findings

1. **The defect-escape result replicated at N=5, in both directions.** On
   `multifile-notify-digest`, every unstructured run - 5/5 in arm A, 5/5 in arm B - shipped
   a digest flush that ignores quiet hours (SPEC R5) and declared done on green self-written
   tests; the held-back suite caught all ten. The mandated-planning arm escaped in 2/5 runs.
   Counts, not significance: at n=5 the 5/5-vs-2/5 difference has a one-sided Fisher exact
   p of 0.083 - direction consistent with the re-spike, but below conventional
   significance. Arm A's runs again declined the pipeline on these tickets and behaved
   identically to the baseline, run after run.

2. **A bad plan propagates with authority - the new failure mode, and this run's most
   important finding.** Arm R's two escapes (runs n2, n3) were not the process being
   skipped; they were the process executing faithfully on a wrong premise. In both runs the
   planner misread SPEC R5 and wrote ACs pinning the *opposite* quiet-hours behaviour
   (digest flush exempt from quiet hours), and the critic - which reviews the
   implementation against the story's ACs - approved, because the implementation matched
   the wrong oracle. That planner-plus-critic failure reproduced twice (2/5). How far the
   error then propagated differed: in n2 the worker went on to edit the workspace's own
   `docs/SPEC.md`, adding clauses stating quiet hours never defer digests - falsifying the
   source of truth and poisoning the evidence trail for every later reader (once, not
   twice: n3's SPEC.md is byte-identical to the fixture original; there the wrong rule
   lived in the worker's code comments and visible tests only). Model tiers cannot explain
   any of this: both escaping deliveries ran on medium, the same claude-sonnet-5 that arms
   A/B were pinned to. Three of five planners read the same spec correctly and their runs
   passed 8/8 hidden tests. The exposed gap is that no step independently checks the
   planner's ACs against the source spec - the critic inherits the plan's oracle. That is
   a concrete, fixable design gap (an AC-vs-spec conformance review before
   implementation), not an argument against planning: the unstructured arms shipped the
   same defect ten times out of ten.

3. **Auditability separated the arms again - and tracked the escapes exactly.** On
   notify-digest: R 0.88 > A 0.68 > B 0.60. Arm R's three clean runs each scored 1.0; its
   two escaping runs scored 0.6 and 0.8, so within-arm the audit score identified precisely
   the runs that shipped the defect (both auditors honestly reported the quiet-hours
   evidence unanswerable - n2's citing the falsified spec, n3's the mis-commented code and
   its visible test). Ordering caveat versus the re-spike: A
   edged above B here (0.68 vs 0.60) where the n=1 re-spike had B above A - the A-B gap is
   noise; the R gap is the signal. On ledger-drift scores were near-ceiling for all arms
   (A 1.00, B 0.97, R 0.97; the two 0.833 rows were single Class-T misses - one stitched
   quote failing the verbatim check, one fact mismatch).

4. **Routing cut arm R's delivery cost 60% on the easy fixture, with zero quality cost.**
   On ledger-drift the planner routed 4/5 deliveries to tiny (haiku) and 1/5 to medium
   (a difficulty score of 23 fell in the low band, whose default tier is small; small is
   undeclared in the routing *model map* - distinct from the summary's *price map*, which
   prices all five tiers - so it degraded upward to medium, the sparse-map rule working as
   designed). Mean cost index 0.40 versus 1.0 for an all-medium mix; the tiny-tier runs
   produced 0/4 escapes and 0.96 mean auditability. On notify-digest every planner picked
   medium - the router correctly refused to down-tier the harder task.

5. **Cost: the full routed pipeline runs ~3.1x baseline tokens per single ticket**
   (179,945/57,860 = 3.11x on notify-digest; 141,672/44,343 = 3.19x on ledger-drift) and
   2.4-3.1x wall time. Phase split (mean per run): planner ~57-67k, worker ~40-59k, critic
   ~45-54k - the overhead is the two extra passes, not a slower implementation. Per
   protocol this buys defect protection and evidence quality, not speed; the fan-out
   amortisation claim remains unmeasured here.

## Caveats

- **n=5 per cell.** Escape counts are reportable but not conventionally significant
  (finding 1); auditability and token spreads are tight, escape behaviour is the noisy
  dimension.
- **One fixture pair, authored in-ecosystem** (independently fairness-reviewed;
  everything needed to pass is present in each visible workspace). The notify-digest
  defect class - a spec rule silently interacting with the ticket - carries all of the
  escape signal.
- **Planner-vs-critic contributions remain partially entangled** in arm R, though this run
  sharpened the picture: when the planner is right the critic adds assurance, and when the
  planner is wrong the critic (as currently scoped) follows the plan. All 10 critic
  verdicts were APPROVE, including both escaping runs.
- Arm A vs R still differ by instruction (judgement-scaled vs mandated), so R-vs-B remains
  the cleaner pipeline-value comparison; A-vs-B measures the default posture.

## Verdict against the pre-registered questions

**Does the mandated pipeline reduce defect escapes on under-specified multi-file work?**
Directionally yes, replicated (10/10 unstructured escapes vs 2/5 mandated), below formal
significance at this n. **Does it correctly scale down?** Yes - Tier 2 control unchanged.
**Is auditability measurable and honest?** Yes - it graded the tool's own failures down,
within-arm and between-arm. **Does routing save cost without quality loss?** Yes on the
easy fixture (0.40 index, no escapes); it declined to save on the hard one, correctly.
**New:** the "bad plan propagates" failure mode is now documented - the
planner-mis-pins-AC, critic-approves-wrong-oracle pattern reproduced twice; its worst
consequence, the worker falsifying the workspace spec, occurred once - and it points at a
specific pipeline change (independent AC-vs-spec check before implementation) rather than
a vague aspiration.
