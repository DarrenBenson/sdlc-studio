# v4 RC rerun of protocol v2 - Tier 1, three model eras

> **Status:** measured · **Date:** 2026-07-10 · **Protocol:** [protocol-v2.md](protocol-v2.md)
> (unchanged: same frozen fixtures, hidden suites, arms, and metrics) ·
> **What changed since the [2026-07-08 N=5 run](2026-07-08-n5-run.md):** the skill payload is
> the v4.0.0-rc (team generation, consult quota, plan-review gate) and the delivery model is
> disclosed per era below. The two datasets are NOT one series - the model generation is a
> confound this report keeps in the open, and it turns out to be the story.

## What was run

Tier 1 (`multifile-notify-digest`, `change-request-ledger-drift`), arms A (pipeline,
judgement retained), B (baseline, skill environmentally absent), R (routed pipeline).
Every workspace received the v4 RC payload where the protocol places it. 72 measured
runs, each scored by its held-back suite, recorded in `tools/bench/results/runs.jsonl`
(`run_id` prefixes `v4-` Fable 5, `v4s-` Sonnet 5, `v4o-` Opus 4.8 bridge):

- **Fable 5:** 2 fixtures x 3 arms x 5 runs = 30
- **Sonnet 5** (the realistic base model, per the operator): 30
- **Opus 4.8** bridge: 2 fixtures x arms A/B x 3 runs = 12

Two additional measures: the protocol's auditability quiz (Sonnet n1 cells, graded
mechanically by `audit_quiz.py`), and a **post-hoc solution-quality rubric** - added at
the operator's request during the run, so it is exploratory and NOT pre-registered. One
reviewer agent per cell scored every run 1-5 on five anchored dimensions (spec fidelity
incl. rule interactions, test quality, design fit, spec/doc maintenance, scope
discipline); reviewers saw workspaces, not verdicts. Blinding is imperfect (an arm-A
workspace visibly contains pipeline artifacts).

## Findings

1. **On the current frontier model (Fable 5) the traps do not bite: 30/30 clean in every
   arm.** July's escape-prevention story is a statement about the models of its day, not
   a permanent property. Where the model alone clears the bar, the pipeline's measurable
   value on a single ticket is the artifact trail, not defect prevention - and the
   overhead for it is ~1.4x, not 3.1x.

2. **On the models people actually deploy, the hidden-requirement trap still bites hard -
   and the pipeline arm gave no protection, because its agents skipped the pipeline.**
   Sonnet digest: pipeline 5/5 escapes vs baseline 2/5; Opus digest: 3/3 and 3/3. Every
   Sonnet and Opus arm-A agent invoked the sanctioned "scale to ticket size" judgement,
   declared the ticket too small for ceremony, and implemented directly - one Opus agent
   even named quiet-hours-at-flush "the one genuinely ambiguous edge" and still shipped
   without resolving it. This is the July founding observation reproduced a generation
   later: **judgement-gated process converges on the model's unaided behaviour exactly
   when the model most needs the process.** On Fable, by contrast, all ten arm-A runs
   engaged the pipeline (CR/story artifacts, TDD, verify_ac) unprompted.

3. **The actionable product finding: the engagement threshold must not be left to the
   weaker model's own judgement.** July's protection came from the MANDATED planning
   pass; protocol v2's arms deliberately leave judgement in, and this rerun shows the
   judgement failing model-dependently. Filed as a v4.1 CR: an engagement floor
   (multi-file change in a spec-bearing repo trips a mandatory planning/AC pass), which a
   future protocol revision should measure as its own arm.

4. **The routed arm barely routed.** Sonnet/Opus routed agents mostly delivered directly
   after consulting `route.py` (low-confidence recommendations on bare tickets); the one
   run that did delegate to an Opus worker still shipped the quiet-hours defect and its
   Sonnet reviewer approved it - independent review against the same misread spec does
   not catch a shared misreading, the exact plan-propagation failure mode July named.

5. **v4-feature attribution, stated plainly (AC3):** no behaviour change in any arm was
   attributable to the v4 features themselves. The generated team and plan-review gate
   were never exercised - no fixture run generated a team or filed a plan-review - and
   the one review that did occur (the delegated run above) shows the consult objection
   quota's failure surface without its mechanics: the reviewer raised no objection to
   the un-pinned quiet-hours interaction, exactly the favourable-and-vague pattern the
   quota exists to catch, but the quota's templates were not in the review path these
   bare-ticket workspaces used. The v4 payload neither helped nor hurt relative to the
   pre-v4 payload on these fixtures; the model era explains everything measured.

6. **Honest caveats.** Same two fixtures as July carry all signal; n=5 (n=3 Opus) per
   cell; Fable ran as both orchestrator and (mostly) delivery model; the rubric is
   post-hoc and imperfectly blinded; two oracle rows are a phrasing artefact of the
   frozen spec-check; audit sample is n=1 per Sonnet cell. One recording correction is on the record:
   the ledger-drift/A/v4o-n2 row was initially written with `defect_escape: false` by
   orchestrator error (the score had printed false/failed); it was corrected to `true`
   minutes later, before any aggregation - the row's verdict itself was never in doubt.
   Raw rows: `runs.jsonl`; rubric and audit JSON archived with the run
   (`tools/bench/results/v4-rerun/`).

## Exploratory addendum: the mandated arm, recreated on the base models

Finding 2 left the decisive cell unmeasured: what does the base model do when the
planning pass is MANDATORY rather than judged? Five additional Sonnet runs on the
digest fixture answered it (run_id `v4sm-`, recorded `phase: calibration` so the
pre-registered aggregates stay clean; the arm prompt requires a written spec-delta
with per-rule interaction analysis and acceptance criteria BEFORE any code):

- **Escapes: 1/5** - against 5/5 for judgement-gated Sonnet arm A, 4/5 routed, and
  2/5 for the baseline on the same fixture. All five runs produced the planning
  artefact; four resolved the quiet-hours interaction correctly and passed.
- **The one escape is July's named failure mode reproduced exactly:** the run's
  planning pass surfaced the R5 interaction and then resolved it wrongly ("digest
  flush deliberately ignores quiet hours"), documented the wrong call in its
  acceptance criteria, implemented it faithfully, and shipped - a bad plan
  propagating with authority. Mandated planning moves WHERE the error must occur;
  it does not make errors impossible, which is why the plan-review gate exists.
- **Cost: mean 79.2k tokens** - 1.18x the Sonnet baseline (67.1k) and far below
  frontier pricing for a near-frontier outcome (Fable: 0/5 at frontier rates).

**The same arm on Opus 4.8** (run_id `v4om-`, five runs, same mandated prompt):
**escapes 0/5** - against 3/3 in BOTH original Opus arms - at a mean 57.9k tokens,
just 1.07x the Opus baseline (54.0k). All five produced the planning artefact and
all five resolved the quiet-hours interaction correctly at flush time.

The full quadrant on the trap fixture, escapes per runs:

| | Judgement-gated pipeline | Baseline (vibe) | Mandated planning |
| --- | --- | --- | --- |
| Sonnet 5 | 5/5 | 2/5 | 1/5 |
| Opus 4.8 | 3/3 | 3/3 | 0/5 |
| Fable 5 | 0/5 | 0/5 | (not run - unaided is already clean) |

Read together with the measured matrix: on both base models, the same skill in the
same workspace went from worst-in-class under judgement-gated engagement to
best-in-class under mandate, by changing one thing - whether engagement is the
model's decision. This is the direct evidence for the engagement-floor CR and the
strongest measured support for the cheap-model-plus-enforced-process thesis.
Caveats: n=5 per cell, one fixture, not pre-registered (the arm was designed after
the matrix results were known) - protocol v3 should carry it as a first-class arm.

vibe coding does not produce at any price. Spending on a bigger model WITHOUT the
process (Opus vibe) was the worst value measured: costlier than Sonnet-mandated and
defective every time.

## What this means for the published claims

- docs/why and the README keep the July numbers AS July numbers (dated, previous model
  generation) and now cite this rerun for the current-generation picture.
- The strongest current-generation, single-ticket case for the pipeline is: the
  auditability and artifact trail at ~1.1-1.5x tokens, deterministic gates that hold
  whatever the model, and - on the base models most teams run - the finding that
  process-when-judged-necessary is not process-when-needed, together with the
  addendum's converse: mandated process took Sonnet from 5/5 escapes to 1/5 at 1.18x
  tokens and Opus from 3/3 to 0/5 at 1.07x. The fan-out economics remain field-reported, not benchmarked.
