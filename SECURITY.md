# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it via GitHub Issues.

For sensitive issues, use GitHub's private security advisory feature:
**Security** → **Advisories** → **New draft security advisory**

## Response Timeline

- **Initial response:** Within 7 days
- **Status update:** Within 14 days
- **Resolution target:** Within 90 days

## Scope

This policy applies to the SDLC Studio skill and its templates.

## Supported Versions

Support tracks the current major: the latest major receives fixes of every kind, the
previous major receives security fixes only, and anything older is unsupported. Restated
at each major release.

| Version | Supported |
| --- | --- |
| 4.x | Yes |
| 3.x | Security fixes only |
| < 3.0 | No |
